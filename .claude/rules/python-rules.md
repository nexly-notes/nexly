# Python Coding Rules & Style Guide

## Core Rules

1. Always use snake_case for variable names
2. Use only single line comments `#`
3. Use semantic and idiomatic code.
4. Use clear and descriptive names for variables, functions, and classes.
5. Prefer readability over performance and complexity. Write code that is easy to understand and maintain.
6. Use UPPER_SNAKE_CASE for constants.
7. Use kebab-case for filenames and directories.
8. Keep It DRY (Don't Repeat Yourself).
9. Use type hints for all functions and variables.
10. Avoid redundancy and Keep the code concise
